1. CORRIGIR ERRO DE HOOKS EM ProjectDetail.tsx
====================================================================

Problema:
O componente ProjectDetail está renderizando hooks de forma condicional. Provavelmente existe algum return antes do useState/useMemo/useEffect, causando o erro:
“Rendered more hooks than during the previous render”.

Correção obrigatória:
- Todos os hooks devem ficar no topo do componente, antes de qualquer return condicional.
- Não chamar useState, useEffect, useMemo ou hooks customizados dentro de if, loops ou depois de returns condicionais.
- Mover:
  const [previewError, setPreviewError] = useState(false);
  para o topo do componente, junto com os outros hooks.
- Se houver project inexistente, loading ou erro, o return condicional deve acontecer somente depois de todos os hooks.

Estrutura esperada:

function ProjectDetail() {
  const params = ...
  const [previewError, setPreviewError] = useState(false);

  const project = ...
  const galleryImages = useMemo(() => {

  }, [project?.galleryImages]);

  const metrics = useMemo(() => {
    return project?.metricsSummary?.split('|').map(...).filter(Boolean) || [];
  }, [project?.metricsSummary]);

  if (!project) {
    return <NotFound />;
  }

  return (...);
}
