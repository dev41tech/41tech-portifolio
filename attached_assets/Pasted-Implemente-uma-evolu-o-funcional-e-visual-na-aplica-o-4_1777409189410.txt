1. MÍDIA NO ADMIN PARA PROJETOS
====================================================================

Hoje o admin só permite cadastrar texto nos projetos. Adicione suporte para imagens e prévias visuais.

No cadastro/edição de projetos, adicionar campos:

- cover_image_url
  Campo para URL da imagem principal/capa do projeto.

- thumbnail_url
  Campo para URL da imagem menor usada nos cards.

- gallery_images
  Campo para múltiplas URLs de imagens, podendo ser uma lista separada por quebra de linha ou vírgula.

- demo_url
  Campo para link de demonstração do projeto.

- repository_url
  Campo para link do repositório, caso seja público.

- category
  Campo select com opções:
  - Sistema Web
  - BI & Dados
  - Automação
  - Integração
  - IA
  - Infra & Deploy
  - ERP
  - Outro

- metrics_summary
  Campo para destacar resultado em formato curto.
  Exemplo:
  “-60% tempo operacional | 100% centralização | Dados em tempo real”

Regras:
- Se cover_image_url existir, usar na página de detalhe do projeto.
- Se thumbnail_url existir, usar nos cards.
- Se não houver imagem, manter fallback premium com gradiente/ícone.
- Nunca quebrar a interface se uma URL estiver vazia.
- Exibir preview da imagem dentro do modal/formulário admin quando a URL for preenchida.

2. HERO COM VÍDEO CONFIGURÁVEL VIA ADMIN
====================================================================

Hoje o hero permite vídeo, mas não existe controle no painel admin.

Criar uma nova área no admin chamada:

/admin-41tech/settings

Também adicionar item “Configurações” no menu lateral do admin.

Criar configuração geral do site com os campos:

- hero_video_url
  URL do vídeo usado no fundo do hero.

- hero_video_enabled
  Toggle true/false para ativar ou desativar vídeo.

- hero_fallback_image_url
  Imagem usada caso o vídeo não carregue.

- whatsapp_url
  Link principal de WhatsApp/contato.

- contact_email
  E-mail principal de contato.

- linkedin_url
  Link da página do LinkedIn, caso exista.

- cta_primary_label
  Texto do botão principal.
  Valor padrão: “Quero automatizar minha operação”

- cta_secondary_label
  Texto do botão secundário.
  Valor padrão: “Ver projetos”

Comportamento:
- Na home, se hero_video_enabled for true e hero_video_url existir, renderizar vídeo em background com:
  autoplay
  muted
  loop
  playsInline
  preload metadata
- Aplicar blur leve, overlay escuro e gradiente azul/preto sobre o vídeo.
- Se o vídeo falhar ou estiver desativado, usar fallback visual com hero_fallback_image_url ou gradiente premium.
- O admin deve conseguir alterar essas configurações sem mexer no código.

9. DOCKERFILE OBRIGATÓRIO PARA VPS
====================================================================

O projeto deve conter um Dockerfile funcional na raiz.

Objetivo:
Permitir build e execução do projeto Next.js em VPS via Docker/EasyPanel.

Criar/ajustar:

- Dockerfile
- .dockerignore
- docker-compose.yml opcional, mas recomendado
- README com instruções de deploy

Dockerfile deve:
- Usar Node LTS Alpine.
- Instalar dependências.
- Rodar build.
- Rodar app em produção.
- Expor porta 3000.
- Usar npm run start ou comando equivalente.
- Funcionar com Next.js.

Exemplo de requisitos:
- Multi-stage build.
- Não copiar node_modules local.
- Usar variáveis de ambiente em runtime.
- Não expor secrets no Dockerfile.

Criar .dockerignore com:
node_modules
.next
.git
.env
.env.local
npm-debug.log
README.md opcional

No README, documentar:

Como rodar local:
npm install
npm run dev

Como buildar:
npm run build

Como rodar com Docker:
docker build -t 41tech-portfolio .
docker run -p 3000:3000 --env-file .env 41tech-portfolio

Variáveis esperadas no .env.example:
DATABASE_URL=
ADMIN_EMAIL=
ADMIN_PASSWORD=
JWT_SECRET=
NEXT_PUBLIC_SITE_URL=
NEXT_PUBLIC_WHATSAPP_URL=
